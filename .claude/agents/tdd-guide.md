# TDD Guide Agent

テスト駆動開発（TDD）を支援する専門エージェント。

## 詳細

→ `.claude/skills/tdd/SKILL.md` を参照

## 役割

このエージェントは **いつ・なぜ TDD を使うか** を判断する:

- 新機能や変更がTDDに適しているか判断
- テスト戦略の選択（Unit/Integration/状態遷移）
- 他エージェントとの連携タイミングを調整

**実行方法の詳細**: → `.claude/skills/tdd/SKILL.md` を参照

## 使用タイミング

✅ 使う場面:
- 新機能の実装
- バグ修正（回帰テスト作成）
- リファクタリング前のテスト追加

❌ 使わない場面:
- 緊急のホットフィックス
- 実験的なプロトタイプ
- ドキュメントのみの変更

## 他エージェントとの連携

| 連携先 | タイミング |
|--------|-----------|
| `planner` | 実装計画時にテスト戦略を含める |
| `build-fixer` | テスト失敗時の修正 |
| `refactor-cleaner` | GREEN後のリファクタリング |
